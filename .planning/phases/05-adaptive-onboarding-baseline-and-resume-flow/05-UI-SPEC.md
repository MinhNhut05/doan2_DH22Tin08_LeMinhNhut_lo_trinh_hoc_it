## Phase UI Structure

### Route and layout contract

- Route stays `/onboarding`; URL does not change between rounds. Source: D-14.
- Page uses a centered single-column shell with `max-width: 720px`.
- Page padding: `16px` mobile, `24px` tablet+, top padding `32px`, bottom padding `48px`.
- Background remains dark and immersive using existing deep/base/surface tokens rather than introducing a new light onboarding theme.
- Stepper appears at the top of the shell on every onboarding state except a fatal load error.
- Main vertical order:
  1. Stepper
  2. Optional WelcomeBackCard
  3. Round content or RecommendationPanel
  4. Inline error/validation message nearest the action that failed

### Component inventory

| Component | Purpose | Required states |
|-----------|---------|-----------------|
| `Stepper` | 3-step progress indicator for rounds 1-3 | upcoming, current, completed |
| `WelcomeBackCard` | Resume CTA for users returning mid-flow | visible, hidden, loading |
| `QuestionCard` | Shared answer UI shell | default, hover, selected, disabled, validation error |
| `RoundOne` | Existing baseline profile questions | loading, ready, submit pending, submit error |
| `RoundTwo` | Career direction + learning style questions | ready, submit pending, sequential gate error |
| `RoundThree` | Dynamic skill self-assessment topics | loading topics, ready, submit pending, submit error |
| `RecommendationPanel` | Recommendation summary + confirm action | loading, ready, confirm pending, confirm error |

## Component Contracts

### 1. `Stepper`
- Horizontal 3-step control above the content card stack.
- Each step uses a `24px` circular marker connected by a `2px` line.
- State styling:
  - upcoming: `dp-surface` fill, `dp-border-subtle` outline, muted text
  - current: accent fill, white text, medium glow
  - completed: accent fill with check icon or filled number, white text
- Labels sit below each marker in `14px` label text.
- Progress track underlay may use `Progress` to show `33%`, `66%`, or `100%`.
- On mobile, keep all 3 steps visible without horizontal scroll.

### 2. `WelcomeBackCard`
- Appears only when backend status says resume is available and recommendation is not yet confirmed.
- Visual style: one `Card` using existing secondary surface plus subtle border.
- Content order: heading, short body, completed-round summary, CTA.
- CTA is full width on mobile, intrinsic width on tablet+.
- No prior answers are shown. Source: D-12.

### 3. `QuestionCard`
- One question per card.
- Card padding: `24px`; question-to-options gap: `16px`; option-to-option gap: `8px`.
- Option row min height: `44px`.
- Single-choice uses `RadioGroup`; multi-choice uses `Checkbox` rows.
- Selected option state must apply all of the following:
  - accent border
  - subtle accent background tint
  - text promoted to primary
  - no layout shift between unselected and selected states
- Validation error appears below the card in destructive color, not as a toast.

### 4. `RoundOne`, `RoundTwo`, `RoundThree`
- All round renderers share the same header structure: display title, short helper copy, card stack, bottom CTA.
- Round content changes inside one motion container; page shell does not remount.
- Rounds remain strictly sequential; if server denies access, show sequential gate message and route user back to the permitted round state.
- Round 3 topic cards are generated dynamically from backend status/context, but visual structure stays fixed.

### 5. `RecommendationPanel`
- Primary recommendation is the top card and visually dominant.
- Content order:
  1. title
  2. source badge
  3. short reason paragraph
  4. focus area chips
  5. alternative path chips
  6. confirmation CTA
- CTA label is `Xác nhận lộ trình`.
- Confirm action remains disabled only during pending mutation, not after a transient error.
- On success, UI may show a short success state, but navigation happens only after backend confirmation is successful. Source: D-17.

## State and Interaction Contract

### Server-derived states
- Onboarding status is authoritative from the backend, not from `isNewUser` or local storage. Source: 05-RESEARCH.md.
- Resume behavior:
  - completed rounds `[1]` => show WelcomeBackCard and start round 2 after CTA
  - completed rounds `[1,2]` => show WelcomeBackCard and start round 3 after CTA
  - no completed rounds => start round 1 immediately, no WelcomeBackCard
- Do not show a dashboard redirect between rounds. Source: D-06.

### Validation and error placement
- Required-answer validation appears inline under the affected question card before submit.
- Submission/network errors appear in one inline `Alert` above the primary CTA for the active round.
- Fatal load failure may use a full-page compact error card pattern aligned with `/home/minhnhut_dev/projects/path-learn/frontend/src/components/feedback/PageError.tsx`.

### Loading states
- Initial status load: skeleton for stepper and one skeleton content card.
- Round submit: keep current answers visible, disable CTA, replace CTA label with loading text.
- Recommendation load after round 3: show Stepper in completed state and a dedicated recommendation loading card; do not flash back to question state.
- Confirm submit: disable confirm CTA, preserve recommendation card content.

### Accessibility rules
- Every question group needs a visible question title and programmatic label.
- Selection state must be visible without relying on color only; include check/radio indicator change plus border/background change.
- Focus-visible uses the existing global outline token from `/home/minhnhut_dev/projects/path-learn/frontend/src/index.css`.
- Motion must respect `prefers-reduced-motion`; when reduced motion is active, use opacity-only changes with near-zero duration.

## Motion Contract

- Library: `framer-motion`. Source: D-16 and 05-RESEARCH.md.
- Use `AnimatePresence` with one motion container wrapping round content.
- Standard transition for round changes:
  - enter: `opacity 0 -> 1`, `x 16 -> 0`
  - exit: `opacity 1 -> 0`, `x 0 -> -16`
  - duration: `0.30s`
  - easing: existing `smooth` curve (`cubic-bezier(0.16, 1, 0.3, 1)`)
- Resume-card reveal uses fade-in only, `0.20s`.
- Do not animate layout height with springy bounce in this phase; keep transitions calm and instructional.
