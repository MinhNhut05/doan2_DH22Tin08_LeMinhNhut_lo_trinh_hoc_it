# Plan 06-03: Dashboard API — Main Path Prioritization and Pending Round

## Goal
Update the dashboard backend to return explicit `mainPath`, `secondaryPaths`, and `pendingOnboardingRound` fields instead of a flat `enrolledPaths` array. Update the frontend hook type to match the new shape.

## must_haves
- Dashboard API returns `mainPath` object (or null) separate from `secondaryPaths` array
- Dashboard API returns `pendingOnboardingRound` (roundNumber or null)
- Frontend DashboardData type matches the new backend shape
- Main path includes: id, name, slug, progress, currentLesson, totalLessons, completedLessons
- Secondary paths include: id, name, slug, progress, currentLesson compact

## Tasks

<task id="01">
<title>Update DashboardService.getOverview to return mainPath and secondaryPaths</title>
<read_first>
  - backend/src/modules/dashboard/dashboard.service.ts
  - backend/prisma/schema.prisma
  - backend/src/modules/learner-profile/learner-profile.service.ts
</read_first>
<action>
In `backend/src/modules/dashboard/dashboard.service.ts`:

1. Add to the Phase 1 parallel queries array a query for the learner profile to get mainLearningPathId:
```typescript
// Add as 6th query in Promise.all:
this.prisma.learnerProfile.findUnique({
  where: { userId },
  select: { mainLearningPathId: true },
}),
```

2. Add a 7th query for pending onboarding round:
```typescript
this.prisma.pendingOnboardingRound.findFirst({
  where: {
    userId,
    dismissCount: { lt: 3 },
  },
  select: { roundNumber: true, dismissCount: true },
  orderBy: { roundNumber: 'asc' },
}),
```

3. Update the destructuring to include `learnerProfile` and `pendingRound`.

4. After Phase 2 (pathsWithProgress), split the result into mainPath and secondaryPaths:

```typescript
const mainLearningPathId = learnerProfile?.mainLearningPathId ?? null;

const mainPath = mainLearningPathId
  ? pathsWithProgress.find(p => p.id === mainLearningPathId) ?? null
  : pathsWithProgress[0] ?? null;

const secondaryPaths = pathsWithProgress.filter(
  p => p.id !== (mainPath?.id ?? ''),
);
```

5. Replace `enrolledPaths: pathsWithProgress` in the return object with:
```typescript
mainPath,
secondaryPaths,
pendingOnboardingRound: pendingRound
  ? { roundNumber: pendingRound.roundNumber }
  : null,
```
</action>
<acceptance_criteria>
  - `dashboard.service.ts` contains `mainLearningPathId: true` in a select clause
  - `dashboard.service.ts` contains `pendingOnboardingRound.findFirst`
  - `dashboard.service.ts` contains `dismissCount: { lt: 3 }`
  - `dashboard.service.ts` contains `mainPath,` in the return object
  - `dashboard.service.ts` contains `secondaryPaths,` in the return object
  - `dashboard.service.ts` contains `pendingOnboardingRound:` in the return object
  - `dashboard.service.ts` does NOT contain `enrolledPaths: pathsWithProgress` in the return
</acceptance_criteria>
</task>
