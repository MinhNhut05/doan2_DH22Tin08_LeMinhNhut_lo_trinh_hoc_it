# Plan 06-10: Settings Main Path Switch UI

## Goal
Add a "Lo trinh chinh" section to the Settings page where the user can see their current main learning path and switch to another enrolled path. Create a dedicated `useSwitchMainPath` mutation hook that calls `PATCH /learner-profile/main-path` and invalidates related queries on success.

## must_haves
- Settings page shows a "Lo trinh chinh" section with current main path name
- "Doi lo trinh" button opens a list of enrolled paths to select from
- Confirm selection calls PATCH /learner-profile/main-path endpoint
- After success, invalidate dashboard + learner-profile + learning-paths queries
- Vietnamese strings added to vi.ts for all new UI copy
- useSwitchMainPath hook uses useMutation with proper invalidation

Create `frontend/src/hooks/useSwitchMainPath.ts`:

```typescript
import { useMutation, useQueryClient } from '@tanstack/react-query';

import api from '../services/api';

export function useSwitchMainPath() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: async (learningPathId: string) => {
      const res = await api.patch('/learner-profile/main-path', { learningPathId });
      return res.data.data;
    },
    onSuccess: async () => {
      await Promise.all([
        queryClient.invalidateQueries({ queryKey: ['dashboard'] }),
        queryClient.invalidateQueries({ queryKey: ['learner-profile', 'me'] }),
        queryClient.invalidateQueries({ queryKey: ['learning-paths'] }),
      ]);
    },
  });
}
```

Key points:
- Follows the same pattern as `useConfirmOnboardingPath.ts` (useMutation + queryClient invalidation)
- Calls `PATCH /learner-profile/main-path` with `{ learningPathId }` body (matching 06-04 SwitchMainPathDto)
- On success: invalidates `dashboard`, `learner-profile`, and `learning-paths` queries so all related UI reflects the change
- Does NOT navigate anywhere — the user stays on Settings page
