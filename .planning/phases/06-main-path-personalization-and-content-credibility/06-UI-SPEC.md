## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge icon spacing |
| sm | 8px | Compact element spacing, stacked metadata, chip gaps |
| md | 16px | Default element spacing, card internals, form field gaps |
| lg | 24px | Section padding, vertical rhythm inside major cards |
| xl | 32px | Layout gaps between dashboard sections |
| 2xl | 48px | Major section breaks, onboarding card separation |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: icon-only interactive targets are minimum 44x44px; primary CTAs, banner CTA, and path-switch confirm actions are minimum 44px tall.

### Layout rules for this phase
- Onboarding recommendation stack: 16px gap between cards.
- Dashboard top-level sections: 24px vertical gap.
- Main path card internal spacing: 24px.
- Secondary path list items: 16px internal padding, 12-16px gap between rows.
- Settings switch dialog/list: 16px between selectable rows, 24px between title/body/actions.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

### Type usage rules
- Use 28px display only for page or panel-leading titles such as the recommendation screen heading.
- Use 20px heading for section titles: main path card title, settings section title, recommendation card title.
- Use 16px body for explanations, descriptive copy, and dialog body text.
- Use 14px label for badges, metadata labels, helper text, and compact CTA context.
- Allowed weights in this phase: 400 and 600 only.
- Do not introduce additional font sizes or font weights in Phase 06 UI.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#050506` / `#0A0A0F` | App background, page backdrop, large canvas surfaces |
| Secondary (30%) | `#0F0F18` / `#161622` | Cards, sidebar, nav, dialog surfaces, grouped containers |
| Accent (10%) | `#3B82F6` | Selected recommendation card, primary CTA, main-path progress/focus indicators, active selection |
| Destructive | `#EF4444` | Destructive actions only |

Accent reserved for: selected recommendation border/background treatment, “Gợi ý hàng đầu” badge, primary buttons, active radio/selection state in path switcher, progress emphasis for the current main path, and focus-visible outlines. Do not use accent for every clickable element.

### Color behavior rules
- Top recommendation card gets the strongest accent treatment: accent border, subtle accent background tint, and one accent badge.
- Secondary recommendation cards stay on secondary surfaces with neutral borders.
- Secondary path cards on Dashboard must remain visually quieter than the main path card.
- Destructive red is reserved for dismiss/error/destructive messaging only. Do not use red for ordinary warnings or path switching.
- Continue using existing success/warning tokens for status badges only when content meaning requires it.

| Element | Copy |
|---------|------|
| Primary CTA | Chọn lộ trình này |
| Empty state heading | Chưa có dữ liệu phù hợp |
| Empty state body | Nếu chưa tạo được gợi ý: “Chưa thể tạo gợi ý lúc này. Vui lòng thử lại sau.” Nếu chưa có lộ trình phụ: “Bạn chưa có lộ trình phụ nào. Hãy tiếp tục với lộ trình chính hoặc khám phá thêm.” |
| Error state | Không thể cập nhật lộ trình lúc này. Hãy thử lại sau hoặc tải lại trang. |
| Destructive confirmation | Đổi lộ trình chính: “Đổi lộ trình chính sang {tên lộ trình}? Tiến độ học trước đây vẫn được giữ nguyên.” Ẩn banner cập nhật hồ sơ: không cần modal; đóng ngay bằng nút “X”. |

### Phase-specific locked copy
- Recommendation section title: `Gợi ý lộ trình cho bạn`
- Top ranked badge: `Gợi ý hàng đầu`
- Dashboard pending-round banner title: `Đã đến lúc cập nhật hồ sơ!`
- Dashboard pending-round banner CTA: `Cập nhật ngay`
- Settings section title: `Lộ trình chính`
- Settings switch trigger: `Đổi lộ trình`
- Secondary paths section title: `Các lộ trình khác`
- Secondary path row CTA: `Tiếp tục`

### Tone rules
- Vietnamese first, natural, concise, and confidence-building.
- Recommendation explanations must reference learner data directly, but remain readable in 2-3 sentences.
- Avoid generic AI phrasing such as “Dựa trên dữ liệu của bạn” when a more specific explanation can be shown.
- Switching-path copy must explicitly reassure that learning history is preserved.

## Phase-Specific Visual and Interaction Contract

### 1. Ranked recommendation cards
- Show exactly 3 recommendation cards.
- Cards are vertically stacked in rank order 1 → 3 inside the existing onboarding shell width.
- Each card must include, in this order:
  1. Rank label
  2. Path name
  3. Difficulty badge
  4. Match score as a percentage
  5. Profile-linked explanation in 2-3 sentences
  6. Optional focus-area chips if backend returns them
  7. Full-width primary CTA: `Chọn lộ trình này`
- The top card must be visually distinct with accent emphasis and `Gợi ý hàng đầu` badge.
- Cards ranked 2 and 3 must remain selectable, but visually secondary.
- Selecting a card immediately submits that `learningPathId`. No extra confirm step on the onboarding screen.
- While submit is pending, disable all three CTAs to prevent double submission.

### 2. Recommendation card states
- Loading: use one container-level loading card, not three fake placeholder cards.
- Error: show inline alert above or within the recommendation container using the locked error copy.
- Empty/fallback unavailable: use the locked empty-state copy; do not render blank card shells.
- Success after selection: route transition can occur immediately after backend confirms success.

### 3. Dashboard main-path prioritization
- The main path card is the most prominent learner-content surface on the Dashboard.
- It appears above secondary paths and above any optional supporting modules.
- Required content in the main path card:
  - path icon or visual marker
  - path name
  - progress percentage
  - current/next lesson title
  - lessons completed / total lessons
  - progress bar
  - primary CTA to continue the next lesson
- Main path CTA is the only filled primary button in the learning-path area.
- If no next lesson is available, keep the button disabled and use neutral wording; do not show a broken link.

### 4. Secondary path treatment
- Secondary paths render in a separate section titled `Các lộ trình khác`.
- Each secondary path row/card is smaller and less visually dominant than the main path card.
- Secondary entries may show:
  - path name
  - compact progress
  - simple `Tiếp tục` text CTA
- Secondary rows use secondary surfaces and neutral borders only.
- Do not use accent fills on secondary path cards unless a row is keyboard-focused.

### 5. Milestone-triggered onboarding banner
- Show the pending-round banner at the top of the Dashboard content area when backend returns a pending round.
- Banner is non-blocking and never uses a modal or forced redirect.
- Banner content structure:
  - title: `Đã đến lúc cập nhật hồ sơ!`
  - one short body sentence explaining profile refinement
  - primary CTA: `Cập nhật ngay`
  - dismiss button: `X`
- Banner visual hierarchy:
  - secondary surface container
  - subtle accent border or accent left edge
  - one filled primary CTA
  - dismiss button stays visually secondary
- Dismiss action is immediate, no confirm dialog.
- After 3 dismissals for the same round, the banner must stop appearing.

### 6. Settings path-switch contract
- Add a dedicated section titled `Lộ trình chính` on Settings/Profile.
- Show current main path name before the switch action.
- `Đổi lộ trình` opens a controlled selection UI containing enrolled paths only.
- Path options use radio-style single selection. Exactly one path can be selected.
- Confirming a new main path requires a confirmation dialog/message using the locked preservation copy.
- Path switch does not present any destructive warning about data loss because no data is lost.
- After successful switch, refresh dashboard-related queries and reflect the new main path immediately.

### 7. AI chat personalization surface
- No new large AI chat layout is required for Phase 06.
- If the chat header exposes current learner context, use a small muted label/chip only.
- Main path context in chat is informational, not promotional.
- Do not add a second CTA that competes with the main learning-path CTA.

### 8. Content credibility contract
- Recommendation explanations may reference only real, published, backend-authoritative path content.
- Do not promise lesson outcomes that the Frontend path does not yet support.
- The recommended Frontend path must feel coherent through the first 5 lessons and first completed track.
- Placeholder lesson references, fake counts, and speculative curriculum copy are not allowed in user-facing UI.

## Accessibility and Interaction States

- All interactive controls must remain keyboard reachable and visible with the project focus-visible outline.
- Minimum touch target: 44x44px for dismiss buttons, icon-only buttons, and CTA controls.
- Recommendation card order must match DOM order and visual order.
- Match score must not be color-only; always render as text percentage.
- Top recommendation state must not rely on color alone; include the `Gợi ý hàng đầu` label.
- Path switch options must be fully operable via keyboard using radio-group semantics.
- Error alerts and banner updates should be announced with appropriate semantic containers; do not hide important feedback in decorative text.
- Respect existing reduced-motion behavior. Recommendation and dashboard transitions may fade/slide, but must become effectively static when reduced motion is enabled.
