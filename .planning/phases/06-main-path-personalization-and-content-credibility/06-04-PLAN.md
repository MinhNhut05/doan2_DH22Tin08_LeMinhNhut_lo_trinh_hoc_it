# Plan 06-04: Main Path Switch API and Milestone Detection Backend

## Goal
Add a `PATCH /learner-profile/main-path` endpoint to switch the user's main learning path. Add synchronous milestone detection in LearnerProfileService.recalculate that creates PendingOnboardingRound rows when the user completes 5 lessons (round 4) or 1 full track (round 5). Add a `POST /dashboard/dismiss-pending-round` endpoint.

> **Wave note:** This plan is Wave 2a (runs after 06-03) because both 06-03 and 06-04 modify `dashboard.service.ts` and `dashboard.controller.ts`. Sequential execution avoids merge conflicts on those files. Plan 06-04 also modifies `learner-profile.service.ts` which 06-05 depends on, but 06-05 already has its own dependency chain (06-01, 06-02) and adds new methods rather than modifying existing ones.

## must_haves
- PATCH /learner-profile/main-path switches mainLearningPathId to a provided learningPathId (must be enrolled)
- Switch does NOT delete UserProgress or QuizResult history
- Milestone detection at 5 completed lessons creates PendingOnboardingRound for round 4
- Milestone detection at 1 completed track creates PendingOnboardingRound for round 5
- POST /dashboard/dismiss-pending-round increments dismissCount (max 3)

```typescript
async switchMainPath(userId: string, learningPathId: string): Promise<void> {
  // Verify user is enrolled in this path
  const enrollment = await this.prisma.userLearningPath.findUnique({
    where: { userId_learningPathId: { userId, learningPathId } },
  });

  if (!enrollment) {
    throw new NotFoundException('You are not enrolled in this learning path');
  }

  // Verify profile exists
  const profile = await this.prisma.learnerProfile.findUnique({
    where: { userId },
  });

  if (!profile) {
    throw new NotFoundException('Learner profile not initialized');
  }

  // Update mainLearningPathId — does NOT touch UserProgress or QuizResult
  await this.prisma.learnerProfile.update({
    where: { userId },
    data: { mainLearningPathId: learningPathId },
  });
}
```

3. Add PATCH endpoint in `LearnerProfileController`:

```typescript
@Patch('main-path')
@UseGuards(JwtAuthGuard)
@HttpCode(HttpStatus.OK)
async switchMainPath(@Req() req: Request, @Body() dto: SwitchMainPathDto) {
  const user = req.user as JwtUser;
  return this.learnerProfileService.switchMainPath(user.id, dto.learningPathId);
}
```

```typescript
/**
 * Creates a PendingOnboardingRound if:
 * - The round hasn't already been completed (exists in OnboardingRound)
 * - The pending record doesn't already exist
 */
private async createPendingRoundIfNeeded(userId: string, roundNumber: number): Promise<void> {
  // Check if round already completed
  const completed = await this.prisma.onboardingRound.findUnique({
    where: { userId_roundNumber: { userId, roundNumber } },
  });
  if (completed) return;

  // Upsert to avoid duplicate pending records
  await this.prisma.pendingOnboardingRound.upsert({
    where: { userId_roundNumber: { userId, roundNumber } },
    create: { userId, roundNumber },
    update: {},  // no-op if already exists
  });
}
```

```typescript
async dismissPendingRound(userId: string, roundNumber: number): Promise<void> {
  const pending = await this.prisma.pendingOnboardingRound.findUnique({
    where: { userId_roundNumber: { userId, roundNumber } },
  });

  if (!pending) return; // silently no-op if not found

  if (pending.dismissCount >= 2) {
    // This is the 3rd dismiss — increment to 3 so it won't show again
    await this.prisma.pendingOnboardingRound.update({
      where: { userId_roundNumber: { userId, roundNumber } },
      data: { dismissCount: 3 },
    });
  } else {
    await this.prisma.pendingOnboardingRound.update({
      where: { userId_roundNumber: { userId, roundNumber } },
      data: { dismissCount: { increment: 1 } },
    });
  }
}
```

2. Add endpoint to `DashboardController`:

```typescript
) {
  const user = req.user as JwtUser;
  return this.dashboardService.dismissPendingRound(user.id, body.roundNumber);
}
```
