# Plan 06-09: Dashboard Frontend UI — Main Path, Secondary Paths, and Pending-Round Banner

## Goal
Refactor the Dashboard page frontend to show the main path prominently at top with a continue-learning card, secondary paths in a "Cac lo trinh khac" section below, and a non-blocking pending-round banner with dismiss functionality. Uses the new API shape (mainPath + secondaryPaths + pendingOnboardingRound) delivered by 06-03.

## must_haves
- Dashboard reads `mainPath` from API response, NOT `enrolledPaths[0]`
- Main path section: continue-learning card with next lesson, progress bar, path name, lessons count
- Secondary paths section titled "Cac lo trinh khac" with compact progress and "Tiep tuc" CTA
- Pending-round banner at top of dashboard content: title, body, CTA, dismiss X
- Dismiss calls backend endpoint and removes banner from UI
- Banner does not appear if pendingOnboardingRound is null
- Vietnamese strings added to vi.ts for all new UI text
- Dashboard query key supports targeted invalidation after dismiss

In `frontend/src/pages/Dashboard.tsx`:

1. Add `X` icon import from lucide-react and `useMutation` + `useQueryClient` imports from @tanstack/react-query.

2. Add a dismiss mutation inside the `Dashboard` component:

```typescript
const queryClient = useQueryClient();
const pendingRound = data?.pendingOnboardingRound ?? null;

const dismissMutation = useMutation({
  mutationFn: async (roundNumber: number) => {
    await api.post(`/onboarding/rounds/${roundNumber}/dismiss`);
  },
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: qk.dashboard });
  },
});
```

3. Add the pending-round banner as the FIRST child inside the `motion.div` wrapper, before the greeting grid:
