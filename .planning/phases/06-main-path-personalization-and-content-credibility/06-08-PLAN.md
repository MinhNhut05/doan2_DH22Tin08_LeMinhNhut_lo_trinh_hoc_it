### Critical Gaps Identified

**Gap 1 — JavaScript Core: scope, closures, `this` (between `javascript-basics` and `es6-features`)**

`javascript-basics` covers variables, data types, control flow, and function declarations. Then `es6-features` jumps directly to arrow functions (which change `this` binding), classes (which need `this`), and destructuring. A beginner who doesn't understand function scope vs block scope, closures, and the `this` keyword will not understand WHY arrow functions exist or HOW classes differ from constructor functions. This gap also blocks React comprehension: hooks rely on closures, event handlers rely on `this`/binding context.

Fix: Add `javascript-scope-closures` lesson at JS Core track order 2.

**Gap 2 — JavaScript Core: array higher-order methods (between `es6-features` and `dom-manipulation`)**

`es6-features` covers syntax (arrow functions, destructuring, spread/rest) but NOT the array methods that USE that syntax daily: `map`, `filter`, `reduce`, `find`, `some`, `every`. These are critical for React list rendering (`items.map(item => <Card />)`), data transformation, and functional programming patterns used throughout the React ecosystem. Without this lesson, learners hit `dom-manipulation` with event delegation patterns but without the data-processing vocabulary they'll need in every React component.

Fix: Add `javascript-array-methods` lesson at JS Core track order 4.
