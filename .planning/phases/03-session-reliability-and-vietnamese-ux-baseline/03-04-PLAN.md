# Plan 04: Vietnamese String Migration for Remaining Screens

Migrate all remaining user-facing screens to use centralized `vi.ts` strings with proper Vietnamese diacritics. Plan 03 covers Dashboard, Explore, Lesson, Quiz, and LessonSidebar. This plan covers Login, AuthCallback, Settings, Onboarding, AiChat, Landing, Sidebar, and AppLayout.

## must_haves

- Login.tsx imports and uses `vi.auth.*` strings instead of hardcoded Vietnamese
- Settings.tsx imports and uses `vi.settings.*` strings instead of hardcoded Vietnamese
- Onboarding.tsx imports and uses `vi.onboarding.*` strings instead of hardcoded Vietnamese
- AiChat.tsx imports and uses `vi.aiChat.*` strings instead of hardcoded Vietnamese
- Sidebar.tsx imports and uses `vi.sidebar.*` strings instead of hardcoded English/Vietnamese
- AuthCallback.tsx uses `vi.auth.callbackLoading` instead of hardcoded text
- All migrated files have zero bare ASCII Vietnamese (no missing diacritics)
- All migrated files have zero user-facing English labels in learner-facing UI
